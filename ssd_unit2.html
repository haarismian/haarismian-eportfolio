Question 1

Create a 2-column multi-line table. In the left-hand column, include the software development stages of the Scrum agile life cycle approach to project management. In the right-hand column, describe the processes which you recommend are applied at each stage to ensure that secure software is produced at the end of the development. To support the preparation of your response, you can refer to the following literature: Sharma, A. & Bawa, R. K. (2020) Identification and Integration of Security Activities for Secure Agile Development. International Journal of Information Technology.

Response

Software Development Stages Recommended Secure Software Processes
1. Pre-requirement
  • Initial education on secure development practices.
  • Redesign of the internal development process to incorporate security considerations.
  • Ensure compliance with existing security frameworks and standards.
2. Requirement Phase
  • Establish role matrix to define responsibilities and access controls.
  • Define security requirements alongside functional requirements.
  • Agree on definitions of security terms and concepts.
  • Identify resources and establish trust boundaries.
  • Specify the operational environment for the software.
  • Implement adaptive monitoring and abuse case scenarios.
3. Design
  • Develop a security architecture that aligns with the system design.
  • Apply secure design principles to address security requirements.
  • Document assumptions and potential security risks.
  • Conduct thorough requirements inspection with a focus on security aspects.
  • Perform risk analyses to identify and prioritize critical assets.
  • Establish quality gates and define acceptance criteria based on security considerations.
  • Measure security based on risk indicators and establish risk metrics.
4. Implementation
  • Apply coding rules and best practices for secure coding.
  • Utilize security tools, such as automated acceptance and unit tests, and conduct static code analyses.
  • Encourage pair programming for code review and knowledge sharing.
5. Testing
  • Perform dynamic analysis to identify runtime vulnerabilities.
  • Conduct comprehensive security testing, including vulnerability and penetration testing.
6. Release
  • Plan for operational readiness, including security considerations.
  • Implement code signing for integrity and authenticity.
  • Develop an incident response plan for security breaches.
  • Continuously improve the security of the software repository.
  • Perform a final security review to ensure all necessary measures have been implemented.

Question 2

Some say that people are the biggest risk of cyber security. Select five terms from ISO/IEC Standard 27000 Section 3 Terms and Definitions and write a 300-word blog post on how people can be managed to overcome cyber security attacks from the inside.

Response

Cybersecurity threats can originate from within an organization, making people the first line of defense and, unfortunately, a potential vulnerability. To mitigate the risks of internal cybersecurity attacks, organizations must focus on effectively managing their human resources. In this blog post, we will explore five key aspects—access control, authentication, confidentiality, integrity, and availability—and discuss how organizations can manage people to overcome cybersecurity attacks from within. 1. Access Control: Implementing robust access control mechanisms is essential to restrict unauthorized access to sensitive systems and data. By employing principles like the "least privilege" approach and role-based access control (RBAC), organizations can ensure that individuals have access only to the resources necessary to perform their duties. Regular reviews of access privileges and prompt removal of unnecessary permissions help minimize the risk of insider threats.

2. Authentication: Authentication serves as a critical gatekeeper in preventing unauthorized access. By implementing strong authentication methods such as multi-factor authentication (MFA), organizations can significantly enhance the security of user identities. MFA combines multiple factors, such as passwords, biometrics, or security tokens, to provide an additional layer of assurance. Educating employees about the importance of strong authentication and enforcing secure password practices further strengthens the authentication process.

3. Confidentiality: Protecting the confidentiality of sensitive information is paramount. Organizations should establish clear guidelines regarding data handling and classification. Training employees on data protection measures, such as encrypting sensitive data and utilizing secure communication channels, helps prevent unauthorized disclosure. By fostering a culture of confidentiality and ensuring employees understand their responsibilities, organizations can mitigate the risk of insider data breaches.

4. Integrity: Maintaining the integrity of data and systems is crucial to prevent unauthorized modifications or tampering. Organizations should enforce strict change control processes, ensuring that only authorized individuals can make changes. By implementing file integrity monitoring systems, regular vulnerability assessments, and robust change management procedures, organizations can detect and address any unauthorized alterations promptly.

5. Availability: Ensuring the availability of critical systems and resources is essential. Organizations should implement redundancy measures, backup protocols, and disaster recovery plans to minimize downtime in the event of an attack. Conducting regular training and simulations to educate employees on incident response and business continuity processes helps maintain operational resilience.

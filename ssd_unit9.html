Learning Objectives

    • Understand and apply CRUD (Create, Read, Update, Delete) capabilities in Python for effective software development.
    • Expand knowledge on Python libraries, specifically focusing on the Flask library for creating APIs.
    • Reflect on and design an ontology to support distributed architecture in software systems.
    • Create an API to perform file management and develop user interfaces to access the back-end code.
    • Enhance communication skills by engaging in peer responses, offering constructive feedback and creating content that encourages reflection and response.

Reflections of the activity and course learning objectives

As an aspiring software engineer, the contents of this chapter will deeply influence my career trajectory and capabilities. Understanding and applying CRUD operations using Python will empower me to efficiently interact with databases, enabling me to manage data in applications using tools such as SQLAlchemy or Django ORM, integral parts of modern software development. Mastering API creation with Python's Flask library will not only allow me to create robust backend services but also position me to effectively use or develop RESTful or GraphQL APIs, vital in facilitating interaction between various software components. By embracing distributed systems and understanding the significance of ontologies, I am prepared for the trend towards microservices architecture and cloud computing, making me well-equipped to work with cloud platforms like AWS, Google Cloud, or Azure. Furthermore, the use of ontologies in defining standardised service deployments aligns with the growing emphasis on DevOps practices, where tools such as Docker and Kubernetes are increasingly used for orchestration in distributed environments. Lastly, the emphasis on peer feedback and communication mirrors the Agile methodology prevalent in modern software engineering, bolstering my team collaboration skills for efficient and productive software development processes in real-world environments.

Question 2

Running the command w3m http://127.0.0.1:5000/user/Ann will send a GET request to the Flask API server for the user data of 'Ann'. If the user 'Ann' exists in our users array, it will return the data related to 'Ann', otherwise it will return 'User not found'. In this case Ann does exist

Question 3

Running the command w3m http://127.0.0.1:5000/user/Adam will send a GET request to the Flask API server for the user data of 'Adam'. As 'Adam' does not exist in our users array, it will return 'User not found'. In this case Adam does not exist

Question 4

The Flask library provides the capability to create web servers and develop web applications in Python. It offers tools and libraries to create routes, handle HTTP requests and responses, and develop RESTful APIs. It is lightweight and modular, allowing developers to use extensions for tasks like form validation, upload handling, various open authentication technologies, and several common framework related tools.

Architecture Evolution Activity

In a potential next phase of evolution history, from microservices to the technologies commonly in use today, the focus might shift towards serverless architectures and Function as a Service (FaaS) platforms like AWS Lambda and Google Cloud Functions. These technologies abstract the server even more, allowing developers to focus solely on the functionality of individual pieces of code that are only running when needed, reducing costs and improving scalability. The rise of edge computing, where computation is performed closer to the data source, could also be highlighted as it reduces latency and allows for real-time data processing capabilities.

Unit 9 Portfolio activity: Cyclomatic Complexity

Cyclomatic complexity is a software metric that measures the number of linearly independent paths through a program's source code. It can help in understanding the complexity and maintainability of the code.

To calculate the cyclomatic complexity, we use the formula:

Cyclomatic complexity (CC) = Number of edges (E) - Number of nodes (N) + 2 * Number of connected components (P)

In this code snippet, there is a single method with a simple control flow. There are two decision points (the two if statements), and no loops.

We can calculate the cyclomatic complexity by counting the number of edges, nodes, and connected components.

Number of edges (E): 5

Number of nodes (N): 4

Number of connected components (P): 1 (since it's a single method)

Using the formula, we get:

CC = 5 - 4 + 2 * 1 = 3

The cyclomatic complexity of this code snippet is 3.
